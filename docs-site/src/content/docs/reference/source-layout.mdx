The nx-neptune repository is organized into three main modules:

## nx_neptune

**Core library** — Neptune Analytics integration, algorithm implementations, session management, and instance lifecycle.

[Browse on GitHub →](https://github.com/awslabs/nx-neptune/tree/main/nx_neptune)

```
nx_neptune/
├── __init__.py                  # Package exports
├── na_graph.py                  # NeptuneGraph class (graph sync)
├── interface.py                 # Backend interface registration
├── session_manager.py           # SessionManager API (data lake workflow)
├── instance_management.py       # Instance lifecycle (create/start/stop/delete)
├── algorithms/                  # Algorithm implementations
│   ├── centrality/              # degree_centrality, closeness_centrality
│   ├── communities/             # louvain, label_propagation
│   ├── link_analysis/           # pagerank
│   ├── traversal/               # bfs_edges, bfs_layers, descendants_at_distance
│   └── util/                    # Algorithm utilities
├── clients/                     # AWS service clients
│   ├── na_client.py             # Neptune Analytics API client
│   ├── iam_client.py            # IAM operations
│   ├── opencypher_builder.py    # openCypher query construction
│   ├── neptune_constants.py     # Algorithm and response constants
│   └── na_models.py             # Data models
└── utils/                       # Shared utilities
    ├── decorators.py            # @configure_if_nx_active, task orchestration
    ├── utils.py                 # General helpers
    └── task_future.py           # Async task management
```

### Key files

| File | Purpose |
|------|---------|
| `na_graph.py` | `NeptuneGraph` class — syncs NetworkX graph data to Neptune Analytics |
| `session_manager.py` | `SessionManager` — full data lake lifecycle (create → import → analyze → export → destroy) |
| `instance_management.py` | Programmatic instance create/start/stop/delete with polling |
| `opencypher_builder.py` | Builds openCypher queries for each algorithm |
| `decorators.py` | `@configure_if_nx_active()` decorator that gates algorithm execution |

## nx_plugin

**NetworkX plugin** — entry point that registers nx-neptune as a NetworkX backend.

[Browse on GitHub →](https://github.com/awslabs/nx-neptune/tree/main/nx_plugin)

```
nx_plugin/
├── __init__.py    # Backend registration, algorithm function list, additional_docs
└── config.py      # NeptuneConfig class (instance configuration settings)
```

When you `pip install nx_neptune`, the `[project.entry-points."networkx.backends"]` in `pyproject.toml` registers this module so that `backend="neptune"` works automatically.

### Key files

| File | Purpose |
|------|---------|
| `__init__.py` | Declares supported `functions` list and `additional_docs` for NetworkX discovery |
| `config.py` | `NeptuneConfig` — configuration for Neptune Analytics instances (graph ID, region, IAM role) |

## connectors (Athena connectors)

**Java-based Athena federated query connectors** for data sources not natively supported by Athena.

[Browse on GitHub →](https://github.com/awslabs/nx-neptune/tree/main/connectors)

```
connectors/
├── pom.xml                              # Parent POM (multi-module Maven build)
├── aws-athena-query-federation/         # Git submodule (Federation SDK)
│   └── athena-jdbc/                     # JDBC base module
├── athena-databricks-connector/         # Databricks Unity Catalog connector
└── athena-s3vector-connector/           # S3 Vector connector
```

### athena-databricks-connector

JDBC-based connector for querying Databricks Unity Catalog tables via Athena federated queries. Extends the `athena-jdbc` module from the [AWS Athena Query Federation SDK](https://github.com/awslabs/aws-athena-query-federation).

[Browse on GitHub →](https://github.com/awslabs/nx-neptune/tree/main/connectors/athena-databricks-connector)

### athena-s3vector-connector

Lambda-based connector for reading vector embeddings from Amazon S3 Vectors via Athena federated queries.

[Browse on GitHub →](https://github.com/awslabs/nx-neptune/tree/main/connectors/athena-s3vector-connector)

### Building connectors

<Aside type="note">
  The `athena-jdbc` module is **not published to Maven Central**, so it must be built from source via the git submodule.
</Aside>

```bash
# Initialize submodule (first time)
git submodule update --init

# Build all connectors
mvn -f connectors/pom.xml clean package -DskipTests

# Build only Databricks connector
mvn -f connectors/pom.xml package -DskipTests -pl :athena-databricks-connector -am
```
